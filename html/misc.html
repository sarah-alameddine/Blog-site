Fork BOMB 

In comp1521 we were learning about fork(), but my interest really peaked when
my tutor briefly mentioned a fork bomb.

A fork is a system call in Unix and Linux systems that takes an parent(existing) process,
and creates a child (duplicate) process. While the child is a duplicate of the parent,
its memory and ID is its own. This allows processes to carry out unique tasks simultaneously.

A fork bomb is basically a DoS attack which crashes a system, forcing
a user to reboot to get rid of the bomb. So it can used to annoy
the crap out of people. However, this is not the only use, sysadmins use it to test user process limitations on server.
Furthermore, as this attack is very effective in reducing a system's available resources
and making the system inaccessible and unusable, it does have room for malicious use.

One method of preventing fork bombs is to limit the amount of processes each user can open.
For example, if a user was limited to having 40 processes running, the fork bomb would hit
the 40 process limit quickly. Therefore, it would be unable to cause the system to run out of resources.

How it Works? (taken directly from geeksforgeeks)

Fork bombs operate both by consuming CPU time in the process of forking, and by
saturating the operating system’s process table. A basic implementation of a fork
bomb is an infinite loop that repeatedly launches new copies of itself.
To incapacitate a system, they rely on the assumption that the number of programs
and processes which may execute simultaneously on a computer. fork() will generate
new process but if you put this process in while true loop, then it will create many
processes and when the limit is crossed, your system will crash.

How to create a fork bomb?

I looked it online - its quite simple:
You're just creating an infinite loop that keeps calling fork()

    #include <sys/types.h<

    int main()
    {
        while(1)
           fork();
        return 0;
